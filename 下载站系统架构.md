# 十九、项目配置说明

## HMR (热模块替换) 配置

在 `vite.config.ts` 中已禁用 HMR：

```typescript
server: {
  hmr: false, // 禁用以修复视频解析页面刷新问题
}
```

### 为什么禁用 HMR？

在开发过程中发现，启用 HMR 会导致视频解析页面出现异常刷新问题，影响用户体验。禁用后：

* ✅ 视频解析功能稳定
* ✅ 页面状态保持正常
* ⚠️ 开发时需要手动刷新页面查看代码更改

### 影响范围

* 开发环境：代码修改后不会自动热更新，需手动刷新浏览器
* 生产环境：无影响，HMR 仅在开发模式下工作

如需临时启用 HMR 进行调试，可将 `hmr: false` 改为 `hmr: true`，但可能会遇到视频解析页面的刷新问题。

## FFmpeg 本地化优化

音视频合并功能使用 FFmpeg WebAssembly 版本，已将核心文件下载到本地以提升加载速度：

### 文件位置

```
public/ffmpeg/
├── ffmpeg-core.js (112KB)
└── ffmpeg-core.wasm (31MB)
```

### 加载策略

* **延迟加载**：页面打开时不加载，只在用户选择完视频和音频文件后才自动加载
* **本地文件**：从本地 `/ffmpeg` 路径加载，避免 CDN 访问慢的问题
* **快速响应**：本地加载速度远快于从 CDN 下载

### 优势

* ✅ 页面秒开，无需等待大文件下载
* ✅ 不依赖外部 CDN，加载稳定可靠
* ✅ 用户体验更好，加载速度更快

### 更新 FFmpeg

如需更新 FFmpeg 版本，执行以下命令：

```bash
cd public/ffmpeg
curl -L -o ffmpeg-core.js https://unpkg.com/@ffmpeg/core@版本号/dist/umd/ffmpeg-core.js
curl -L -o ffmpeg-core.wasm https://unpkg.com/@ffmpeg/core@版本号/dist/umd/ffmpeg-core.wasm
```

### 已知问题和解决方案

**问题：FFmpeg 初始化卡住**

在某些情况下，FFmpeg 可能在初始化阶段（`ffmpeg.load()`）卡住不动。

**解决方案：**

1. **使用单线程模式**：在 `ffmpeg.load()` 时添加 `workerURL: coreURL` 参数
2. **移除 COOP/COEP 响应头**：这些响应头虽然是 SharedArrayBuffer 的要求，但可能导致兼容性问题
3. **添加超时处理**：使用 `Promise.race()` 添加 30 秒超时，避免无限等待

**当前配置：**

```typescript
await ffmpeg.load({
  coreURL,
  wasmURL,
  workerURL: coreURL, // 单线程模式
});
```

**浏览器兼容性：**

- ✅ Chrome/Edge 90+
- ✅ Firefox 89+
- ✅ Safari 15.2+
- ⚠️ 需要启用 JavaScript 和 WebAssembly
