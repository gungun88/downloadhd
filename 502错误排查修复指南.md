# 502 Bad Gateway 错误排查和修复指南

## 问题分析

错误信息：`POST https://www.downloadhd.net/api/parse 502 (Bad Gateway)`

**502 错误原因：**
- Nginx 配置正确，前端可以访问
- 但 Nginx 无法连接到后端服务（`http://127.0.0.1:8000`）
- 说明后端服务未启动或端口不对

---

## 立即排查步骤

### 1. 检查后端服务是否运行

SSH 登录服务器后，执行：

```bash
# 检查 Python 后端进程
ps aux | grep python

# 检查 8000 端口是否被监听
netstat -tlnp | grep 8000
# 或者
ss -tlnp | grep 8000

# 如果使用 PM2 管理
pm2 list
pm2 status
```

**预期结果：**
- 应该看到 Python 进程在运行
- 8000 端口应该在 LISTEN 状态

---

### 2. 如果后端没有运行，启动后端服务

#### 方法一：使用 Python 直接启动（测试用）

```bash
cd /www/wwwroot/downloadhd.net/backend

# 激活虚拟环境（如果有）
source venv/bin/activate

# 启动后端
python3 main.py
```

#### 方法二：使用 PM2 启动（推荐）

```bash
cd /www/wwwroot/downloadhd.net/backend

# 启动服务
pm2 start main.py --name downloadhd-backend --interpreter python3

# 保存 PM2 配置
pm2 save

# 设置开机自启
pm2 startup
```

#### 方法三：使用 systemd 服务（生产环境推荐）

创建服务文件：

```bash
sudo nano /etc/systemd/system/downloadhd-backend.service
```

内容：

```ini
[Unit]
Description=DownloadHD Backend Service
After=network.target redis.service

[Service]
Type=simple
User=www
Group=www
WorkingDirectory=/www/wwwroot/downloadhd.net/backend
Environment="PATH=/www/wwwroot/downloadhd.net/backend/venv/bin"
ExecStart=/www/wwwroot/downloadhd.net/backend/venv/bin/python3 main.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl start downloadhd-backend
sudo systemctl enable downloadhd-backend
sudo systemctl status downloadhd-backend
```

### 4. 检查 Redis 是否运行

后端可能依赖 Redis，检查 Redis 状态：

```bash
# 检查 Redis 状态
systemctl status redis

# 如果未运行，启动 Redis
systemctl start redis
systemctl enable redis

# 测试 Redis 连接
redis-cli ping
# 应该返回 PONG
```

---

### 5. 检查后端配置文件

```bash
cd /www/wwwroot/downloadhd.net/backend
cat .env
```

确保配置正确：

```env
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_DB=0
API_BASE_URL=https://downloadhd.net
PORT=8000
```

---

### 6. 测试后端是否可访问

```bash
# 在服务器上测试后端
curl http://127.0.0.1:8000/api/health
# 或
curl http://127.0.0.1:8000/

# 应该返回 JSON 响应，而不是连接错误
```

---

### 7. 检查防火墙（如果后端在其他服务器）

如果后端不在本机，检查防火墙：

```bash
# 检查防火墙状态
firewall-cmd --state

# 开放 8000 端口（仅在后端独立服务器时需要）
firewall-cmd --permanent --add-port=8000/tcp
firewall-cmd --reload
```

**注意：** 如果后端和 Nginx 在同一台服务器，不需要开放 8000 端口到外网。

---

### 8. 重启 Nginx

```bash
# 测试 Nginx 配置
nginx -t

# 重启 Nginx
systemctl restart nginx

# 查看 Nginx 错误日志
tail -f /www/wwwlogs/downloadhd.net.error.log
```

## 常见问题和解决方案



```bash

pip install -r requirements.txt
```

## 快速修复脚本

创建一个快速修复脚本：

```bash
nano /www/wwwroot/downloadhd.net/fix-backend.sh
```

内容：

```bash
#!/bin/bash

echo "=== 检查后端服务状态 ==="
cd /www/wwwroot/downloadhd.net/backend

# 检查进程
if pgrep -f "python.*main.py" > /dev/null; then
    echo "✓ 后端进程正在运行"
else
    echo "✗ 后端进程未运行，正在启动..."

    # 激活虚拟环境
    source venv/bin/activate

    # 使用 PM2 启动
    pm2 start main.py --name downloadhd-backend --interpreter python3
    pm2 save

    echo "✓ 后端已启动"
fi

# 检查端口
if netstat -tlnp | grep :8000 > /dev/null; then
    echo "✓ 8000 端口正在监听"
else
    echo "✗ 8000 端口未监听"
fi

# 检查 Redis
if systemctl is-active --quiet redis; then
    echo "✓ Redis 正在运行"
else
    echo "✗ Redis 未运行，正在启动..."
    systemctl start redis
fi

# 测试后端
echo "=== 测试后端连接 ==="
curl -s http://127.0.0.1:8000/ || echo "✗ 后端无法访问"

echo "=== 完成 ==="
```

赋予执行权限并运行：

```bash
chmod +x /www/wwwroot/downloadhd.net/fix-backend.sh
bash /www/wwwroot/downloadhd.net/fix-backend.sh
```
