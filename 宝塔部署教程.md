# 宝塔面板部署教程 - 视频下载工具

## 一、环境准备

### 1.1 服务器要求
- 操作系统：CentOS 7+ / Ubuntu 18+ / Debian 9+
- 内存：至少 2GB RAM
- 硬盘：至少 20GB 可用空间
- 域名：downloadhd.net（已解析到服务器 IP）

### 1.2 安装宝塔面板

```bash
# CentOS 安装命令
yum install -y wget && wget -O install.sh https://download.bt.cn/install/install_6.0.sh && sh install.sh

# Ubuntu/Debian 安装命令
wget -O install.sh https://download.bt.cn/install/install-ubuntu_6.0.sh && sudo bash install.sh
```

安装完成后记录：
- 宝塔面板地址
- 用户名
- 密码

## 二、宝塔面板配置

### 2.1 登录宝塔面板
访问 `http://你的服务器IP:8888`，使用安装时的账号密码登录。

### 2.2 安装软件环境

在宝塔面板 → 软件商店，安装以下软件：

**前端环境：**
- Nginx 1.22+
- Node.js 18+ (使用 PM2 管理)

**后端环境：**
- Python 3.10+
- Redis 7.0+

**其他：**
- SSL 证书管理器

## 三、项目部署

### 3.1 上传项目文件

1. 在宝塔面板 → 文件，创建目录：
   ```
   /www/wwwroot/downloadhd.net
   ```

2. 上传项目文件到该目录，或使用 Git：
   ```bash
   cd /www/wwwroot/downloadhd.net
   git clone 你的仓库地址 .
   ```

### 3.2 前端部署

```bash
cd /www/wwwroot/downloadhd.net

# 安装依赖
npm install

# 构建生产版本
npm run build
```

构建完成后，`dist` 目录包含所有前端静态文件。

### 3.3 后端部署

```bash
cd /www/wwwroot/downloadhd.net/backend

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cp .env.example .env
nano .env  # 编辑配置
```

编辑 `.env` 文件：
```env
redis_url=redis://:你的Redis密码@localhost:6379
```

**重要：** 将 `你的Redis密码` 替换为实际密码。在宝塔面板 → 数据库 → Redis → 配置文件中查看 `requirepass` 的值。

### 3.4 使用 PM2 管理后端进程

**重要：** 必须使用 uvicorn 启动 FastAPI 应用，不能直接运行 main.py。

```bash
cd /www/wwwroot/downloadhd.net/backend

# 激活虚拟环境
source venv/bin/activate

# 安装额外依赖
pip install instaloader python-multipart

# 使用 PM2 启动（使用虚拟环境中的 uvicorn）
pm2 start "venv/bin/uvicorn main:app --host 0.0.0.0 --port 8000" --name downloadhd-backend

# 保存 PM2 配置
pm2 save

# 设置开机自启
pm2 startup
```

验证服务状态：
```bash
pm2 status
pm2 logs downloadhd-backend --lines 20
```

应该看到：
- 状态：`online`
- 日志显示：`Uvicorn running on http://0.0.0.0:8000`

## 四、Nginx 配置

### 4.1 创建网站

1. 宝塔面板 → 网站 → 添加站点
   - 域名：`downloadhd.net`
   - 根目录：`/www/wwwroot/downloadhd.net/dist`
   - PHP 版本：纯静态

### 4.2 配置 Nginx 反向代理

点击网站 → 设置 → 配置文件，修改为：

```nginx
server
{
    listen 80;
    listen 443 ssl;
    listen 443 quic;
    http2 on;
    server_name downloadhd.net www.downloadhd.net;
    index index.html index.htm default.html;
    root /www/wwwroot/downloadhd.net/dist;

    #CERT-APPLY-CHECK--START
    # 用于SSL证书申请时的文件验证相关配置 -- 请勿删除
    include /www/server/panel/vhost/nginx/well-known/downloadhd.net.conf;
    #CERT-APPLY-CHECK--END

    #SSL-START SSL相关配置，请勿删除或修改下一行带注释的404规则
    #error_page 404/404.html;

    #HTTP_TO_HTTPS_START
    set $isRedcert 1;
    if ($server_port != 443) {
        set $isRedcert 2;
    }
    if ( $uri ~ /\.well-known/ ) {
        set $isRedcert 1;
    }
    if ($isRedcert != 1) {
        rewrite ^(/.*)$ https://$host$1 permanent;
    }
    #HTTP_TO_HTTPS_END

    ssl_certificate    /www/server/panel/vhost/cert/downloadhd.net/fullchain.pem;
    ssl_certificate_key    /www/server/panel/vhost/cert/downloadhd.net/privkey.pem;
    ssl_protocols TLSv1.1 TLSv1.2 TLSv1.3;
    ssl_ciphers EECDH+CHACHA20:EECDH+CHACHA20-draft:EECDH+AES128:RSA+AES128:EECDH+AES256:RSA+AES256:EECDH+3DES:RSA+3DES:!MD5;
    ssl_prefer_server_ciphers on;
    ssl_session_tickets on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;
    add_header Strict-Transport-Security "max-age=31536000";
    add_header Alt-Svc 'quic=":443"; h3=":443"; h3-29=":443"; h3-27=":443";h3-25=":443"; h3-T050=":443"; h3-Q050=":443";h3-Q049=":443";h3-Q048=":443"; h3-Q046=":443"; h3-Q043=":443"';
    error_page 497  https://$host$request_uri;
    #SSL-END

    #ERROR-PAGE-START  错误页配置，可以注释、删除或修改
    error_page 404 /404.html;
    #error_page 502 /502.html;
    #ERROR-PAGE-END

    #禁止访问的文件或目录
    location ~ ^/(\.user.ini|\.htaccess|\.git|\.env|\.svn|\.project|LICENSE|README.md)
    {
        return 404;
    }

    #一键申请SSL证书验证目录相关设置
    location ~ \.well-known{
        allow all;
    }

    #禁止在证书验证目录放入敏感文件
    if ( $uri ~ "^/\.well-known/.*\.(php|jsp|py|js|css|lua|ts|go|zip|tar\.gz|rar|7z|sql|bak)$" ) {
        return 403;
    }

    # API 反向代理（重要：必须在静态文件规则之前）
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 300s;
        proxy_send_timeout 300s;
        proxy_read_timeout 300s;
    }

    # 静态文件缓存 - 图片
    location ~ .*\.(gif|jpg|jpeg|png|bmp|swf|ico|svg)$
    {
        expires      30d;
        add_header Cache-Control "public, immutable";
        error_log /dev/null;
        access_log /dev/null;
    }

    # 静态文件缓存 - JS/CSS/字体
    location ~ .*\.(js|css|woff|woff2|ttf|eot)?$
    {
        expires      12h;
        add_header Cache-Control "public";
        error_log /dev/null;
        access_log /dev/null;
    }

    # robots.txt 和 sitemap.xml
    location = /robots.txt {
        try_files $uri =404;
    }

    location = /sitemap.xml {
        try_files $uri =404;
    }

    # 前端路由支持（SPA）- 必须放在最后
    location / {
        try_files $uri $uri/ /index.html;
        # 安全头
        add_header X-Frame-Options "SAMEORIGIN" always;
        add_header X-Content-Type-Options "nosniff" always;
        add_header X-XSS-Protection "1; mode=block" always;
    }

    access_log  /www/wwwlogs/downloadhd.net.log;
    error_log  /www/wwwlogs/downloadhd.net.error.log;
}
```

### 4.3 配置说明

**重要配置项解释：**

1. **HTTP/2 和 QUIC 支持**
   - `http2 on` - 启用 HTTP/2 协议
   - `listen 443 quic` - 启用 HTTP/3 (QUIC) 协议

2. **强制 HTTPS 跳转**
   - 已配置自动跳转，访问 HTTP 会自动跳转到 HTTPS

3. **API 反向代理**
   - `/api/` 路径会转发到后端服务 `http://127.0.0.1:8000`
   - 超时时间设置为 300 秒（5 分钟）

4. **静态文件缓存**
   - 图片文件缓存 30 天
   - JS/CSS 文件缓存 12 小时

5. **SPA 路由支持**
   - `try_files $uri $uri/ /index.html` 确保前端路由正常工作

6. **安全配置**
   - 禁止访问 `.env`、`.git` 等敏感文件
   - 添加安全响应头

### 4.4 SSL 证书说明

你的 SSL 证书已经配置完成，证书路径：
- 证书文件：`/www/server/panel/vhost/cert/downloadhd.net/fullchain.pem`
- 私钥文件：`/www/server/panel/vhost/cert/downloadhd.net/privkey.pem`

宝塔面板会自动续期 Let's Encrypt 证书（每 90 天）。

### 6.1 验证网站所有权

1. 访问 [Google Search Console](https://search.google.com/search-console)
2. 点击"添加资源" → "网址前缀"
3. 输入：`https://downloadhd.net`
4. 选择验证方式：

## 九、常见问题

### 9.1 502 Bad Gateway 错误排查

如果访问网站出现 502 错误，按以下步骤排查：

**1. 检查后端服务状态**
```bash
pm2 status
```
- 如果状态是 `errored`，查看日志：`pm2 logs downloadhd-backend --lines 50`
- 如果状态是 `stopped`，重启服务：`pm2 restart downloadhd-backend`

**2. 检查端口是否正确**
```bash
netstat -tlnp | grep 8000
```
应该看到后端监听在 8000 端口。

**3. 检查 Nginx 配置**
确认 `proxy_pass` 指向正确的端口：
```bash
grep "proxy_pass" /www/server/panel/vhost/nginx/downloadhd.net.conf
```
应该显示：`proxy_pass http://127.0.0.1:8000;`

**4. 常见错误及解决方案**

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| `ModuleNotFoundError: No module named 'instaloader'` | 缺少依赖包 | `pip install instaloader python-multipart` |
| `NOAUTH Authentication required` | Redis 密码未配置 | 更新 `.env` 中的 `redis_url` |
| `Extra inputs are not permitted` | `.env` 配置格式错误 | 使用 `redis_url=redis://:密码@localhost:6379` 格式 |
| `Form data requires "python-multipart"` | 缺少表单处理库 | `pip install python-multipart` |

**5. 重启所有服务**
```bash
pm2 restart downloadhd-backend
nginx -s reload
systemctl restart redis
```

### 9.2 Instagram 解析失败

Instagram 需要有效的 cookies 才能解析。如果出现"内容不可用"错误：

**方法1：使用浏览器插件导出 cookies**

1. 安装 Chrome 插件：[Get cookies.txt LOCALLY](https://chromewebstore.google.com/detail/get-cookiestxt-locally/cclelndahbckbenkjhflpdbgdldlbecc)
2. 登录 Instagram 网页版
3. 点击插件图标 → Export → 保存为 `instagram.txt`
4. 在网站的 Cookies 配置页面（`https://downloadhd.net/cookies`）上传

**方法2：使用 yt-dlp 导出**
```bash
cd /www/wwwroot/downloadhd.net/backend
source venv/bin/activate
yt-dlp --cookies-from-browser chrome --cookies cookies/instagram.txt "https://www.instagram.com/"
pm2 restart downloadhd-backend
```

**验证 cookies 是否有效**
```bash
# 查看 cookies 文件
ls -lh /www/wwwroot/downloadhd.net/backend/cookies/instagram.txt

# 测试解析
curl -X POST http://127.0.0.1:8000/api/parse \
  -H "Content-Type: application/json" \
  -d '{"url":"https://www.instagram.com/reel/C8aKxPWRnHd/"}'
```

### 9.3 网站无法访问
- 检查防火墙：宝塔面板 → 安全，放行 80 和 443 端口
- 检查域名解析是否生效：`ping downloadhd.net`
- 检查 Nginx 是否运行：`systemctl status nginx`

### 9.4 API 请求失败
- 检查后端是否运行：`pm2 status`
- 查看后端日志：`pm2 logs downloadhd-backend`
- 检查 Redis 是否运行：`systemctl status redis`

### 9.5 SSL 证书过期
- Let's Encrypt 证书有效期 90 天
- 宝塔面板会自动续期
- 手动续期：宝塔面板 → 网站 → SSL → 续签

### 9.6 谷歌未收录
- 确认 robots.txt 没有禁止抓取
- 检查 sitemap.xml 格式是否正确
- 在 Google Search Console 查看抓取错误
- 确保网站内容原创且有价值
- 等待时间：新站通常需要 1-4 周
